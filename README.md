# Jaygo Cut

> 面向中文口播视频创作者的 AI 审核剪辑桌面工具。  
> 从转录、审核、AI 标记、精确裁剪，到导出完整剪映草稿，尽量把繁琐的口播粗剪流程做成一个清晰的桌面应用。

## Jaygo Cut 能做什么

Jaygo Cut 主要服务中文口播、访谈、观点类视频的粗剪和审核：

1. 选择视频，自动提取音频并转录。
2. 在审核页查看逐字文本、音频波形和视频预览。
3. 自动标记静音、语气词、重复句和疑似废话。
4. 使用 AI 分析全文，进一步规划需要删除或保留的内容。
5. 支持手动微调、撤销、文本纠错、原文校对。
6. 一键执行精确裁剪，或导出完整剪映草稿继续精修。
7. 可生成发布标题、封面标题、简介、话题参考和插入素材。

## 核心亮点

| 模块 | 亮点 |
| --- | --- |
| 转录引擎 | 支持火山引擎、阿里 DashScope/Qwen ASR、Mimo ASR、本地 Whisper/ggml 等多种方案。 |
| 审核页面 | 逐字文本、波形、视频预览、删除标记、快捷键、撤销、历史恢复联动。 |
| AI 分析 | 按口播剪辑逻辑识别语气词、重复表达、无效铺垫、跑题段落，并尽量保留主题完整性。 |
| AI 管家 | 通过对话执行软件操作，例如重新生成素材、调整删除标记、导出剪映草稿。 |
| 原文校对 | 粘贴口播原文后，按读音和上下文纠正常见识别错误，例如人名、地名、他/她/它。 |
| 剪映草稿导出 | 支持导出完整剪映草稿，包含主视频、字幕、图片素材、视频素材，打开剪映即可继续编辑。 |
| 插入素材 | 可用 LLM 规划分镜，再调用图片/视频生成接口生成配图和视频素材，支持上传替换与重新生成。 |
| 发布建议 | 生成 10 个 30 字内视频标题、5 个封面标题、简介和 #话题参考。 |
| 在线更新 | 支持应用内检查更新、下载更新包和查看更新说明。 |

## 快速开始

1. 打开 Jaygo Cut。
2. 展开“设置与主题”，配置转录引擎和 API Key。
3. 选择视频文件。
4. 点击“开始完整流程”。
5. 转录完成后打开审核页。
6. 检查自动删除标记，必要时使用 AI 分析、原文校对和手动调整。
7. 点击“执行裁剪”导出成片，或点击“导出剪映”生成可编辑草稿。

## 推荐配置

### 转录

- 云端高精度：优先使用火山引擎或阿里 DashScope/Qwen ASR。
- 本地离线：可在设置页检查本机模型，按需安装本地 Whisper/ggml 模型。
- 本地模型不会默认内置在安装包中，避免安装包过大。

### AI 能力

- AI 分析、AI 管家、发布建议、素材规划需要配置兼容 OpenAI 格式的 LLM 服务。
- 插入图片/视频素材需要配置对应图片或视频生成接口。
- 如果模型返回慢或为空，建议降低一次分析的文本长度，或检查服务商并发、RPM、上下文长度限制。

### 剪映草稿

Jaygo Cut 支持三类导出目标：

- 自动放入剪映草稿目录：适合普通用户，导出后打开剪映即可看到草稿。
- 当前项目目录：适合先验证草稿文件结构。
- 自定义草稿根目录或模板目录：适合把 Jaygo Cut 输出放进固定剪映工作区。

模板草稿建议满足：

- 目录中包含剪映可识别的 `draft_content.json` 和 `draft_info.json`。
- 模板最好是明文 JSON 草稿；如果是新版剪映加密草稿，Jaygo Cut 会提示不可用。
- 使用模板前建议关闭剪映，避免草稿文件被锁定。

## 常见问题

### 为什么安装包比较大？

Electron 桌面应用需要内置运行时、前端页面、剪辑脚本和部分工具，因此安装包会比普通脚本大。本地大模型默认不内置，避免进一步增大体积。

### 剪映里提示媒体丢失怎么办？

优先确认：

1. 导出后没有移动或删除 Jaygo Cut 输出目录。
2. 剪映草稿目录有写入权限。
3. 剪映导出时没有被杀毒软件拦截复制媒体文件。
4. 使用模板草稿时，模板不是加密草稿。

### 为什么 AI 标记不一定完全准确？

AI 会根据全文主题、口播结构和用户历史习惯给出建议，但口播剪辑存在主观判断。建议把 AI 标记作为第一轮粗筛，再在审核页快速微调。

### 画质会被压缩吗？

默认导出质量为极致画质，尽量降低压缩损失。不同素材格式、帧率、剪辑模式和插入素材合成策略仍可能影响最终体积和观感。

## 项目定位

Jaygo Cut 不是替代剪映的完整非线编软件，而是把口播视频最耗时的“转录、清理、粗剪、校对、素材规划”前置自动化。最终精修、包装和发布，仍然可以交给剪映等专业编辑器。
